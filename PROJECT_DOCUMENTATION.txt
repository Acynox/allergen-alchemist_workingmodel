PROJECT OVERVIEW
================================================================================
Allergen Alchemist is a web application that helps users with food allergies
find safe recipe alternatives using molecular chemistry analysis. It combines
RecipeDB and FlavorDB APIs to analyze recipes, suggest ingredient substitutes,
generate personalized diet plans, and provide nutritional information.

TECHNOLOGY STACK
================================================================================
Backend:
  - Python 3.x
  - FastAPI (Web framework)
  - Uvicorn (ASGI server)
  - Pydantic (Data validation)
  - Requests (HTTP library)

Frontend:
  - HTML5
  - CSS3 (Custom styling with animations)
  - Vanilla JavaScript (ES6+)
  - Font Awesome (Icons)
  - Google Fonts (Outfit font)

External APIs:
  - RecipeDB API (Recipe and ingredient data)
  - FlavorDB API (Molecular compound data)

-----------------------------------
2. POST /analyze_recipe
-----------------------------------
PURPOSE: Analyze a recipe for allergens and suggest substitutes

REQUEST:
  Method: POST
  URL: http://localhost:8000/analyze_recipe
  Headers: Content-Type: application/json
  Body: {
    "recipe_name": "Apple Pie",
    "user_allergens": ["Egg_Allergy", "Dairy_Allergy"]
  }

RESPONSE:
  Status: 200 OK
  Content-Type: application/json
  Body: {
    "recipe_title": "Grandma's Apple Pie",
    "recipe_id": 12345,
    "found_issues": true,
    "substitutions": [
      {
        "original_ingredient": "butter",
        "substitutes": [
          {
            "name": "coconut oil",
            "score": 0.85,
            "verified": true,
            "common_molecules": ["oleic acid", "palmitic acid"]
          }
        ]
      }
    ]
  }

PROCESSING FLOW:
  1. Receive recipe name and user allergens
  2. Call RecipeDB API to search recipe by title
     ‚Üí GET https://cosylab.iiitd.edu.in/recipedb/recipe-bytitle/recipeByTitle?title={recipe_name}
  3. Extract recipe ID and title from response
  4. Fetch recipe ingredients using recipe ID
     ‚Üí GET https://cosylab.iiitd.edu.in/recipedb/search-recipe/{recipe_id}
  5. For each ingredient:
     a. Check if it matches any user allergens (using bridge.py)
     b. If allergen found, search for substitutes from candidates.json
     c. For each substitute candidate:
        - Fetch molecular compounds from FlavorDB for original ingredient
        - Fetch molecular compounds from FlavorDB for substitute
        - Calculate Jaccard similarity score
        - Find common molecules between them
     d. Sort substitutes by similarity score (highest first)
     e. Take top 3 substitutes
  6. Return all substitutions with molecular synergy data

EXTERNAL API CALLS MADE:
  - RecipeDB: Search recipe by title (1 call)
  - RecipeDB: Get recipe ingredients (1 call)
  - FlavorDB: Get molecules for each ingredient (N calls, where N = ingredients + substitutes)


-----------------------------------
3. POST /generate_diet_plan
-----------------------------------
PURPOSE: Generate a personalized weekly diet plan

REQUEST:
  Method: POST
  URL: http://localhost:8000/generate_diet_plan
  Headers: Content-Type: application/json
  Body: {
    "user_allergens": ["Egg_Allergy"],
    "diet_preference": "both"  // Options: "veg", "non-veg", "both"
  }

RESPONSE:
  Status: 200 OK
  Content-Type: application/json
  Body: {
    "weekly_plan": [
      {
        "day": "Monday",
        "breakfast": "Pancakes",
        "lunch": "Chicken Salad",
        "dinner": "Mushroom Risotto"
      },
      // ... 6 more days
    ]
  }

PROCESSING FLOW:
  1. Receive user allergens and dietary preference
  2. Load predefined recipe pools:
     - Vegetarian recipes (breakfast, lunch, dinner)
     - Non-vegetarian recipes (breakfast, lunch, dinner)
  3. Combine pools based on preference:
     - "veg" ‚Üí Only vegetarian recipes
     - "non-veg" ‚Üí Only non-vegetarian recipes
     - "both" ‚Üí Combined pool
  4. Filter recipes using keyword matching:
     - Check recipe titles for allergen keywords
     - Remove recipes containing allergen keywords
  5. If no safe recipes found, use fallback generic meals
  6. Generate 7-day plan:
     - Randomly select breakfast, lunch, dinner for each day
     - Ensure variety across the week
  7. Return weekly plan

EXTERNAL API CALLS MADE:
  - None (uses keyword-based filtering for performance)

NOTE: Originally used RecipeDB API calls for each recipe, but optimized
      to keyword-based filtering for instant response times.


-----------------------------------
4. POST /get_nutrition
-----------------------------------
PURPOSE: Get nutritional information for a recipe

REQUEST:
  Method: POST
  URL: http://localhost:8000/get_nutrition
  Headers: Content-Type: application/json
  Body: {
    "recipe_name": "Chicken Curry"
  }

RESPONSE:
  Status: 200 OK
  Content-Type: application/json
  Body: {
    "recipe_title": "Chicken Curry",
    "calories": 450,
    "protein": 35,
    "carbs": 42,
    "fat": 18,
    "fiber": 6,
    "sugar": 8
  }

PROCESSING FLOW:
  1. Receive recipe name
  2. Call RecipeDB API to search recipe by title
     ‚Üí GET https://cosylab.iiitd.edu.in/recipedb/recipe-bytitle/recipeByTitle?title={recipe_name}
  3. Extract recipe ID and title
  4. Fetch recipe ingredients
     ‚Üí GET https://cosylab.iiitd.edu.in/recipedb/search-recipe/{recipe_id}
  5. Calculate nutrition based on ingredients:
     - Start with base random values
     - Adjust based on ingredient keywords:
       * Protein-rich: chicken, beef, pork, fish ‚Üí +protein, +calories
       * Carb-rich: rice, pasta, bread, potato ‚Üí +carbs, +calories
       * Fat-rich: oil, butter, cream, cheese ‚Üí +fat, +calories
  6. Return nutritional breakdown

EXTERNAL API CALLS MADE:
  - RecipeDB: Search recipe by title (1 call)
  - RecipeDB: Get recipe ingredients (1 call)

NOTE: Nutrition calculation is estimated based on ingredient types.
      In production, would use a dedicated nutrition API.

-----------------------------------
Jaccard Similarity Calculation
-----------------------------------

PURPOSE: Measure similarity between two ingredients based on shared molecules

FORMULA:
  Jaccard(A, B) = |A ‚à© B| / |A ‚à™ B|

WHERE:
  A = Set of molecules in original ingredient
  B = Set of molecules in substitute ingredient
  ‚à© = Intersection (common molecules)
  ‚à™ = Union (all unique molecules)

EXAMPLE:
  Original: {hexanal, octanal, nonanal}
  Substitute: {hexanal, heptanal, octanal}
  
  Intersection: {hexanal, octanal} ‚Üí 2 molecules
  Union: {hexanal, octanal, nonanal, heptanal} ‚Üí 4 molecules
  
  Jaccard = 2/4 = 0.5 (50% similarity)

IMPLEMENTATION:
  def calculate_jaccard(set1, set2):
      if not set1 and not set2:
          return 0.0
      intersection = len(set1.intersection(set2))
      union = len(set1.union(set2))
      if union == 0:
          return 0.0
      return intersection / union

  3. Values: Allergen category strings
  
EXAMPLE:
  ALLERGEN_MAP = {
      "milk": "Dairy_Allergy",
      "cheese": "Dairy_Allergy",
      "egg": "Egg_Allergy",
      "peanut": "Nut_Allergy",

  }

  def get_allergen_category(ingredient_name):

================================================================================
                        PERFORMANCE OPTIMIZATIONS
================================================================================

1. DIET PLAN GENERATION
   BEFORE: Made RecipeDB API calls for each recipe (slow, 30+ seconds)
   AFTER: Keyword-based filtering (instant, <1 second)
   IMPACT: 30x faster response time

2. PARALLEL PROCESSING
   - Uses ThreadPoolExecutor for concurrent FlavorDB API calls
   - Fetches molecules for multiple substitutes simultaneously
   - Reduces total API call time by ~70%

3. CACHING
   - candidates.json stores pre-fetched ingredient data
   - Avoids repeated API calls for common ingredients
   - Populated by fetch_candidates.py script

4. FRONTEND OPTIMIZATION
   - CSS animations use GPU acceleration (transform, opacity)
   - Minimal DOM manipulation
   - Event delegation where possible
